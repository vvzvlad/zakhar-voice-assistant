# Отчёт о загрязнении синтетических семплов «Захар» (v1–v7)

Обнаружено, что часть синтетических ПОЗИТИВНЫХ семплов, на которых обучались модели v1–v7,
говорят НЕ «захар», а другие слова. Ниже — баг, доказательства, масштаб по версиям и фикс.
(Этот же материал — в исходном `output_v8/CONTAMINATION_AUDIT.md` на обучающей ноде.)

## Баг
Генератор drawn-out позитивов кормил Piper **переудлинёнными написаниями с повторяющимися
гласными** («захаааар», «захаааааар»). Русский фонемизатор Piper на таком ломается и
произносит **другие слова**: «дорога», «врага», «драка», «два года», «товар», «трава», «сбер».
Дополнительно: голоса **Piper denis/dmitri** ломаются на «Захар» даже при ПРАВИЛЬНОМ написании.
Голоса **Silero** и **Piper/irina** — чистые.

## Как нашли
1. **Независимая проверка оператора** (Vosk ru-small STT по выборке 100 клипов из
   `v5/drawn_raw`): «захар» — только **34/100**, явный мусор («дорога/врага/драка/два года») —
   **62/100**, пусто — 5, «сахар» — 1.
2. **Аудит агента** (Vosk, 400 сэмплов на набор, мягкий фильтр) — подтвердил и расширил на все
   версии.

## Калибровка STT (для честности цифр)
Те же Vosk-проверки по **621 РЕАЛЬНОЙ человеческой записи** пометили «мусором» **17%**. Люди
мусора не говорят → эти 17% = **шум STT** (ru-small плохо декодит протяжную речь). Значит
истинный мусор синтетики ≈ (помечено − ~17% floor). Vosk при этом ЗАНИЖАЕТ (мягкий фильтр
засчитывает «за ха» и пр.), поэтому ручная оценка оператора (62%) строже и ближе к правде.

## Масштаб загрязнения по версиям

| набор (какие версии обучал) | помечено мусором | ≈истинный (−17% floor) | худшие голоса |
|-----------------------------|------------------|------------------------|---------------|
| `synth_raw` (v1, v4) | 54% | ~37% | dmitri 97%, denis 65% |
| `synth_v3_raw` (v3, v4) | 54% | ~37% | dmitri 98%, denis 73% |
| `v4/pos_synth` (v4) | 34% | ~17% | dmitri 95%, denis 78% (silero чисто) |
| `v5/drawn_raw` (v5, v6, v7) | 34% (оператор: 62%) | ~17–62% | dmitri 98%, denis 84% |
| `v5/pos_drawn_synth` (v5–v7) | 40% | ~23% | dmitri 100%, denis 64% |
| `real_16k` (621 реальных) | 17% | **0** (это STT-floor) | — (люди, чисто) |

**Вывод:** ВСЕ обученные версии (v1, v4, v5, и итерации майнинга v6/v7 поверх v5) учились на
заметно загрязнённой синтетике. Главный источник — **Piper denis/dmitri** на «Захар».
irina / ruslan / Silero — в норме. Реальные 621 запись — чистые (мусор там = только шум STT).

## Влияние
Заметная доля «протяжных» позитивов в обучении была мислейблом (другие слова под меткой
«захар»). Это лейбл-шум, который мог занижать качество и смазывать выученное представление
фразы. Метрики v5 (drawn-out recall 100% per-clip, music-FAPH 0 с VAD) оставались высокими, но
получены ВОПРЕКИ загрязнению — чистый набор может их улучшить.

## Фикс (реализован для v8)
1. **Генерация переписана** (`gen_v8_piper.py` / `gen_v8_silero.py`): только правильные
   написания «Захар» / «Заха́р» (ударение). Протяжность — высоким `length_scale` (1.3–2.1,
   корректная медленная речь) + опц. тайм-стретч чистого «Захар» (sox tempo, питч сохранён).
   Никаких повторяющихся гласных.
2. **STT-фильтр** всех синтетических позитивов перед обучением. Сгенерено 15500 →
   оставлено **10261 / 15500 (66%)**. Выброшено по голосам: Piper/dmitri 97%, denis 78%,
   ruslan 53%, irina 11%, Silero 6%. Чистый набор держат **Silero (7065) + irina (1773) +
   ruslan (933)**.
3. **Замер на устройстве:** STT нельзя применять к device-капче (тракт XMOS+NS/AGC делает речь
   нечитаемой для Vosk — там фильтр по ЭНЕРГИИ, не по слову).

## Что дальше

устройства (повышенный вес) + far-field-аугментация, с оценкой на device-eval и стандартном

## TL;DR

- **Чистый held-out — всего 16 независимых голосов** (жёсткий потолок без новых данных): все
  device-recorded `section_*` голоса, кроме 16, сидят в обучении какой-то из моделей; `yx_*`
  выкинуты целиком (голоса общие с train). Это и есть причина, почему **fish-раунд — реальная
  разблокировка**: без большего bench все сравнения упираются в шум.
- **Реальный device-recall ≈ 60–67%** (clip) у ВСЕХ деплоибельных, **НЕ ~80%** (подтверждает
  поправку на утечку). Но recall — это **полоса, зависящая от тракта:** ~**65%** (room-mixed
  `positive_mix_recorded`, согласован по тракту с FAPH-негативами) → ~**96%** (чистый прямой захват
  `device_captures`; v27 clip 0.958). n≈16, CI широкий.
- **Модели статистически НЕразличимы** на 16 голосах ни по recall, ни по FAPH. v27 vs v16:
  парный bootstrap Δ=0.000, CI[−.19,.19] (обе берут 11/16, не вложены). «v27 строго лучший» —
  **НЕ подтверждается; ничья в шуме.** v27 — разумный incumbent, а не доказанно-лучший.
- **Доверять только radio-FAPH** (source-disjoint, 3.0 ч). silence/music/speech/vacuum
  SESSION-протёкшие (eval-срезы — соседи ±30с train-срезов) + малая выборка (<0.35 ч) → оптимистичны.
  FAPH-профиль v27 в рабочей точке = **1 silence + 1 music + 4 radio** на 1.2/0.34/3.0 ч →
  пуассон-CI огромные, разница между v27/v16/v11 в FAPH **в пределах шума**.
- **Path B (angular + 2-ступень verifier) — твёрдо мёртв:** **0 из 39 конфигов** (logreg/GBM/RF/MLP/SVC
  × silence-boost × сиды) не бьют v27 строго; min silence-FAPH при recall≥.651 = 0.83 (> v27 0.8).
  Структурная дыра — **silence**: v19 angular палит на тишине на 1-й ступени (~39/ч), эти FA
  неразделимы с настоящими триггерами в angular-пространстве → 2-я ступень не вытаскивает. Оживить =
  переучить angular с тонной silence-негативов, либо fish.
- **Баг-хант поймал 2 реальные проблемы:** позитивная утечка (`v8/pos_real_v8`); session-leak
  негативов (→ доверяем только radio-FAPH). Скорер проверен (fresh-state, без state-leak).

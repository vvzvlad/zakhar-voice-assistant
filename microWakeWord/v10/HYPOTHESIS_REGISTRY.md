# Реестр гипотез/идей «захааар» — проверено / отброшено / не проверено

Каталог ИДЕЙ и их статусов. Ход исследования, описание версий и какой результат у каждой —
в [../README.md](../README.md) и карточках версий (v5/SUMMARY_ALL_VERSIONS, v8/README, v11/README,
v11/SUMMARY_real_eval). Здесь — ТОЛЬКО статусы гипотез, без хроники.

## Статус
Продакшен — **v11** (v8-рецепт + реальные device-tract негативы тишины/музыки). На реальном
device-eval (cutoff 0.9 / win5, VAD): real silence-FAPH 0.8/ч, music 0/ч, FRR на живых людях 0.9%.

## ✅ ПРОВЕРЕНО / СРАБОТАЛО
| идея | результат |
|------|-----------|
| **Реальные device-tract негативы тишины/музыки** (idle-комната+музыка через устройство → hard-негативы) | ФИКС полевого бага: real silence-FAPH 12.5→0.8/ч, music 9.3→0/ч, real-FRR 4.7→0.9% (v11). Синтетика этого не могла |
| **Playback re-recording** (запись через тракт устройства) | главный рычаг recall: FRR 18%→5% (v5→v8) |
| **VAD-прегейт** (ESPHome `vad:`) | инструментальная музыка → 0 FAPH |
| **STT-фильтр синтетики** (Vosk) | убрал 62% мусора (сломанный TTS) |
| **uuid-leakage-safe split + device-eval** | честная шкала; вскрыл, что v5 был регрессом vs v4 |
| **Drawn-out центрирование + рычаг длительности/окна** | держит recall на протяжной фразе |
| **Дедуп позитивов по uuid** | 621→107 уникальных (убраны повторы + утечка) |
| **Confusables с/з** (sampling 8 / penalty 5) | срезали «Сахар»-near-miss |
| **Масштаб негативов** (HF+MUSAN+ESC-50+FMA) | база FAPH |
| **2-я стадия verifier** (on-device логрег 65 пар.) | сам по себе 100% TP / 100% music-FA reject (но angular+verifier как деплой не побил v11 — см. ❌) |
| **QAT vs PTQ** | PTQ уже без потерь (≤0.7pp) → QAT не нужен |
| **Стратифицированный eval** (9 акуст. режимов) | профиль worst-case; harness готов |

## ❌ ОТБРОШЕНО (и почему)
| идея | почему отброшено |
|------|------------------|
| **Настоящий angular-margin head** (AM-Softmax, проверено v10) | сильнейший рычаг RECALL (worst-FRR −13…−16пп), INT8 без потерь — НО растит music-FAPH (≥9/ч vs 0 у v8) → гейт «+recall без роста FAPH» не пройден. Отложено: реальные муз-негативы (s≈10,m=0.2) или + verifier. Тулчейн: [research/am_train.py](research/am_train.py) |
| **Рантайм energy/RMS-гейт перед MWW** (проверено, анализ) | ВРЕДИТ: ложные в тишине AGC-громче слов (−28 vs −37..−72 dBFS) → гейт режет слова, не ложные |
| **SNR-адаптивный / адаптивный порог** (проверено v14) | не чинит высокоуверенные ложные; хуже статичного |
| **Representation/feature-KD** (angular→студент, проверено v15) | хуже по обеим осям; angular-recall не переносится через эмбеддинг-KD |
| **+Yandex-позитивы в обучение** (проверено v11b) | real-FRR ХУЖЕ (6.5% vs 0.9%) — синтетика навредила |
| **Синтетика тишины/шума как негатив** | у обеих моделей 0.0 ложных — баг от РЕАЛЬНОЙ микроструктуры комнаты, синтетикой не воспроизводится |
| **Синтетический music-mix в обучение** | неверное распределение: устройство гонит AEC по loopback → KWS видит ОСТАТОК эха, не полную музыку. Регресс |
| negative_class_weight 30 (v2) | пере-консервативна, recall упал |
| adversarial mining раунды 2–3 | плато |
| Knowledge distillation (logit) | без выигрыша на нашем масштабе (нужно >400k позитивов) |
| Focal loss (γ=2) | регресс |
| margin-BCE (logit-shift аппрокс) | регресс — это НЕ настоящий angular head (тот — в ❌ выше) |
| Residual connections | вредят (0.065→0.137 на тех же данных) |
| SLR28 RIR-банк | слишком гулкий (mit_rirs лучше) |
| Агрессивная multi-condition аугментация | пере-портит (плато ~20×) |
| Тяжёлые SOTA (Keyword Mamba 3.4M, KWT 5M) | не влезают в 32k/45КБ |

## ❓ НЕ ПРОВЕРЕНО (открытые идеи)
| идея | заметка |
|------|---------|
| **Серверная 2-я ступень верификации** | ⚠ блокер: прошивка шлёт на сервер аудио ПОСЛЕ детекции и бипа — самого «захар» сервер не видит. Нужна правка прошивки (пред-триггерный буфер) |
| **Второй (менее обработанный) канал микрофона** ch1 | наш Native-API клиент уже имеет слот `data2`; нужно включить 2-й microphone source в ESPHome YAML + задействовать `data2`. Чисто наш код |
| **Тюнинг PCAN/PCEN-параметров фронтенда** | PCAN усиливает текстуру тишины (корень бага), НО меняет train/serve — фронтенд устройства фиксирован, рискованно |
| **Curriculum multi-condition** (clean→reverb→media по нарастанию) | бьёт uniform-аугментацию (MISP 2021); БЕЗ полногромкого music-mix |
| **Zero-shot voice cloning TTS** (CosyVoice2/XTTS-v2/F5-TTS, рус.) | сотни голосов сверх Yandex; но обычная синтетика уже навредила (v11b) — проверять осторожно |
| **DTW/QbE-шаблон протяжного «захааар»** (2-я ступень) | явная проверка длительности |
| **SSL-учитель (WavLM/Sylber) серверный** | только сервер/обучение, не on-device |
| **BC-ResNet/TC-ResNet порт** | feasibility подтверждена; не сделан |
| **Состязательный тест длительности** (TTS-ряд «захар→захааар», prob vs длина «а») | валидация опоры модели на длительность |
| **Production data-flywheel** (логи поля→псевдометки→дообуч) | задизайнен; ⚠ для майнинга ложных нужен пред-триггерный буфер (прошивка не шлёт) |
| **Sleep-word / abort UX** | отмена случайного срабатывания |

## ⏳ ЖДЁТ DEVICE-СЕМПЛОВ (нужны новые записи через устройство)
- **Реальные позитивы через устройство** (`positive_samples_real_people_recorded`) — агент наиграет колонкой; восстановит dev-FRR и обобщение на незнакомые голоса.
- **Barge-in** (реальный остаток эха через AEC) — синтетикой не воспроизвести.
- **Far-field / muffled / шёпот / дети** — живые записи через устройство.
- Честные метрики добрать (FAPH: реально есть ~4ч тишины + ~55мин музыки; не хватает barge-in/far-field; FRR с N+CI).

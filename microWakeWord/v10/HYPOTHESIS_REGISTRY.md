# Реестр гипотез/идей «захааар» — проверено / отброшено / не проверено

Каталог ИДЕЙ и их статусов. Ход исследования, описание версий и какой результат у каждой —
в [../README.md](../README.md) и карточках версий (v5/SUMMARY_ALL_VERSIONS, v8/README, v11/README,
v16/SUMMARY_real_eval, v16/DURATION_CAUSALITY, v16/VERIFIER_DEPLOY_SPEC). Здесь — ТОЛЬКО статусы
гипотез, без хроники.

## Статус
Продакшен — **v16** (v8-рецепт + реальные device-tract позитивы + 4 реальных негатив-класса:
silence/music/speech/vacuum). На ЧЕСТНОМ device-eval (held-out по источнику, cutoff 0.9/win5):
FRR **21%** (реальный потолок recall стандартной модели — см. duration в ❌), FAPH **0** по всем
классам с VAD → строго лучше v8 (тот давал 12.5/ч в тишине). Прежние «recall 99%/FRR 0.9%» были
оптимистичны (eval по знакомым голосам/одной комнате).
**Полевая рабочая точка: cutoff 0.95** — на 0.80 v16 ложил в РЕАЛЬНОЙ тишине (слышал слово в тихой
комнате); DET v16 плоский (#F/#C) → 0.95 убирает ложные в тишине, держа recall ~21%.

## ✅ ПРОВЕРЕНО / СРАБОТАЛО

| **Реальные device-tract негативы речи/пылесоса** (4-й/5-й класс, v16) | speech/vacuum-FAPH 0 на device-eval; вместе дали FAPH 0 по всем классам с VAD |

| **Voice-safe split (held-out НЕЗНАКОМЫЕ голоса)** (#E) | УТЕЧКИ НЕТ: recall на незнакомых голосах 21.7% ≈ source-сплит 21.0% → прошлые ~21% recall ЧЕСТНЫЕ, клоны не текли (гипотеза speaker-leakage отвергнута) |
| **Drawn-out центрирование + рычаг длительности/окна** | держит recall на протяжной фразе |
| **Дедуп позитивов по uuid** | 621→107 уникальных (убраны повторы + утечка) |
| **Short «захар» hard-негативы → duration-awareness** (v21/v27) | ❌ **АРТЕФАКТ:** «65%→17/23%» было утечкой состояния eval-харнесса (короткие клипы без сброса стрим-состояния). | **detection-aware per-frame head → INT8-стриминг** (Conv1x1, без Flatten) | feasibility подтверждена (конвертится, пер-фреймовый блок); лосс-вариант — см. ❌/❓ |
| **Confusables с/з** (sampling 8 / penalty 5) | срезали «Сахар»-near-miss |
| **Масштаб негативов** (HF+MUSAN+ESC-50+FMA) | база FAPH |

| **Стратифицированный eval** (9 акуст. режимов) | профиль worst-case; harness готов |

## ❌ ОТБРОШЕНО (и почему)
| идея | почему отброшено |
|------|------------------|
| **Реальные device-tract ПОЗИТИВЫ как фикс recall** (v16/v21) | recall НЕ починили: FRR 21% (как v8), val 98.7% vs held-out 21% = gap обобщения (BCE запоминает источники трейна). Рычаг recall — angular или duration-aware короткие-негативы, НЕ объём позитивов. (Позитивы полезны как честная шкала, не для recall) |
| **Опора модели на ДЛИТЕЛЬНОСТЬ гласного** (проверено, nr2/DURATION_CAUSALITY) | НЕТ: модель ловит спектральный ОНСЕТ «заха» → ПАЛИТ на короткий «захар» (@1.0с: v16 65%, v19 93%); хвост гласного ≈0 важности (occlusion/IG), энергия тоже не прокси. «Только протяжный» как фича иллюзорна без short-негативов |
| **angular-margin head как single-model деплой** (v10/v12/v17/v19, с annealing) | сильнейший рычаг RECALL (FRR 3-6%), INT8 стабилен с annealing — НО FAPH НЕ давится: penalty 8/15, cutoff-сатурация, verifier — всё провалилось; палит на короткий «захар»; И КАТАСТРОФА на плотной речи (21.5/ч на radio, #F). Recall через 2-ступень — МЕРТВ (↓); остаётся только duration-aware СТАНДАРТНАЯ |
| **angular + verifier как on-device 2-ступень** (dual-output [kws,verify] + per-trigger AND, v13/v18) | **ЗАКРЫТО (#G):** архитектура+streaming-refit верны (verify на ложных 0.65-0.99→0.01-0.06), больше FA ПОМОГЛО (98→293 radio-FA → held-out reject 25%→~50%), НО verifier всё равно пропускает ~ПОЛОВИНУ незнакомых ложных → FAPH сильно хуже v16. Причина ФУНДАМЕНТАЛЬНА: angular-эмбеддинг ЛИНЕЙНО неразделим (всё «захар-по-онсету» в одном кластере). **Прошивочная опция B мертва** |
| **detection-aware loss pooling — noisy-OR И max-pool** (v23, Apple «Optimize What Matters») | ОБА варианта регресс (noisy-OR FRR 45% — OR раздувает негативы; max-pool тоже хуже базы). Per-frame head конвертится в int8 (✅ feasibility), но лосс в нашем пайплайне не взлетел → на полку |
| **Рантайм energy/RMS-гейт перед MWW** | ВРЕДИТ: ложные в тишине AGC-громче слов (−28 vs −37..−72 dBFS) → гейт режет слова, не ложные |
| **SNR-адаптивный / адаптивный порог** (v14) | не чинит высокоуверенные ложные; хуже статичного |
| **Representation/feature-KD** (angular→студент, v15) | хуже по обеим осям; angular-recall не переносится через эмбеддинг-KD |
| **+Yandex СЫРЫЕ (не-device) позитивы в обучение** (v11b) | real-FRR ХУЖЕ (6.5% vs 0.9%) — НО это было на НЕ-device eval (real_people). device-tract Yandex (проигранные через устройство) — наоборот ПОМОГАЮТ (✅ #E). Вредят именно СЫРЫЕ/не-device, не клоны как таковые |
| **Синтетика тишины/шума как негатив** | у моделей 0.0 ложных — баг от РЕАЛЬНОЙ микроструктуры комнаты, синтетикой не воспроизводится |
| **Синтетический music-mix в обучение** | неверное распределение: устройство гонит AEC по loopback → KWS видит ОСТАТОК эха, не полную музыку. Регресс |
| negative_class_weight 30 (v2) | пере-консервативна, recall упал |
| adversarial mining раунды 2–3 | плато |
| Knowledge distillation (logit) | без выигрыша на нашем масштабе (нужно >400k позитивов) |
| Focal loss (γ=2) | регресс |
| margin-BCE (logit-shift аппрокс) | регресс — это НЕ настоящий angular head |
| Residual connections | вредят (0.065→0.137); RepCNN-репараметризация не пробована — ❓ |
| SLR28 RIR-банк | слишком гулкий (mit_rirs лучше) |
| Агрессивная multi-condition аугментация | пере-портит (плато ~20×) |
| Тяжёлые SOTA (Keyword Mamba 3.4M, KWT 5M) | не влезают в 32k/45КБ |

## ❓ НЕ ПРОВЕРЕНО (открытые идеи)
| идея | заметка |
|------|---------|
| **RepCNN-репараметризация** (обучать multi-branch, инференс single-branch) | residual-выигрыш без рантайм-стоимости (та причина, по которой residuals отброшены); не пробована |
| **Второй (менее обработанный) канал микрофона ch1** | НЕ для beamforming — как РАЗНОСТНЫЙ сигнал для on-device double-talk/self-trigger гейта (ложное = артефакт обработки, по-разному на ch0/ch1). Чисто наш код; ch1-записей пока нет |
| **Серверная 2-я ступень** | блокер СНЯТ (прошивка форкнута, pre-trigger буфер есть — сервер видит «захар»). НО детекцию держим on-device СОЗНАТЕЛЬНО → серверный детектор де-скоупнут (выбор, не блокер) |
| **Тюнинг PCAN/PCEN-параметров фронтенда** | PCAN усиливает текстуру тишины (корень бага), НО меняет train/serve — фронтенд устройства фиксирован, рискованно |
| **Curriculum multi-condition** (clean→reverb→media по нарастанию) | бьёт uniform-аугментацию (MISP 2021); БЕЗ полногромкого music-mix |
| **Zero-shot voice cloning TTS** (CosyVoice2/XTTS-v2/F5-TTS) | сотни голосов; но синтетика-позитивы уже навредили (v11b) — осторожно |
| **DTW/QbE-шаблон протяжного «захааар»** (2-я ступень) | явная проверка длительности |
| **BC-ResNet/TC-ResNet порт** | feasibility подтверждена; не сделан |
| **Production data-flywheel** (логи поля→псевдометки→дообуч) | прошивка форкнута → pre-trigger логирование возможно. ⚠ риск самоотравления (логировать только свои срабатывания → слепые зоны): нужен независимый teacher + фикс. «золотой» held-out + Relative FRR@FAR |
| **Sleep-word / abort UX** | отмена случайного срабатывания |

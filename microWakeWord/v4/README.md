## Вердикт: v4 — основная модель

**v4 строго доминирует v1** по обеим осям на каждом пороге (recall на 93 реальных held-out
↔ music-FAPH на чистой disjoint 90-мин MUSAN-музыке, окно3):

| cutoff | recall v1 → **v4** | music-FAPH (без VAD) v1 → **v4** | music-FAPH (с VAD) |
|-------:|--------------------|----------------------------------|--------------------|
| 0.90 | 0.806 → **0.882** | 8.00 → **2.67** | **0.00** (оба) |
| 0.95 | 0.753 → **0.882** | 6.67 → **2.00** | **0.00** |
| 0.97 | 0.677 → **0.839** | 6.00 → **2.00** | **0.00** |
| 0.99 | 0.591 → **0.753** | 4.00 → **1.33** | **0.00** |

Per-clip recall: v4 mean 0.990, **99% клипов ≥0.9** (v1: 0.945 / 91%).

## Что сработало

1. **Больше спикеров** (рычаг recall): Piper 4 голоса (9200) + **Silero RU** (aidar/baya/
   kseniya/xenia/eugene + `random` = сотни голосов, 6000) + 528 реальных ×5 → recall 99%.
2. **Масштаб и разнообразие негативов** (рычаг музыки): весь HF kahrendt + **MUSAN**
   music/noise + **ESC-50** + fma_medium/small — десятки часов музыки.
3. **Confusable hard-negatives** (сахар/загар/пожар/комар/Захаров/захват/закат/базар/кошмар/
   гусар…), **sampling 8.0 / penalty 5.0** (рецепт malonestar).
4. **Музыко-осознанный отбор весов**: MUSAN-music в `validation_ambient` (split) → трейнер
   минимизирует FAPH, включающий музыку.
5. **Умеренные веса классов** `[30,40,50]`/`[2,2]` (не [60,75]). **Утечка муз-пробника
   починена** (disjoint MUSAN-тест).

## Как ставить

`probability_cutoff 0.95` / `sliding_window_size 3` (манифест), `tensor_arena 45000`
(сверь с живым JSON, можно занизить). **Обязательно `vad:`** в ESPHome — на чистой музыке
снимает music-FAPH в 0.00 без потери recall (см. [model/esphome_micro_wake_word.yaml](model/esphome_micro_wake_word.yaml)).

## Известные лимиты

- **«Сахар»** (с vs з) ещё триггерит ~3% изолированных фраз (v1 ~2%) — фундаментальная
  близость 2-сложной фразы; VAD тут не помогает (это речь). 3–4-сложное слово сняло бы это.
- **Главный рычаг recall дальше — больше РЕАЛЬНЫХ записей** протяжного «Захар» (сбор оператора).
- Не запущено (заложено на следующий раунд): **adversarial-майнинг ложных** и **вторая
  стадия (verifier)** — см. отчёт.

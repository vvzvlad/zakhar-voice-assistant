# fish_audio — генерация синтетических «захааар» через fish.audio

Инструменты для генерации **разнообразных** синтетических примеров слова через
[fish.audio](https://fish.audio) TTS и отбора голосов под это.

## Зачем

Потолок recall (~21% FRR на незнакомых голосах) — это проблема **обобщения**: модель
цепляется за онсет «заха», который специфичен для голоса. Эксперимент #E показал, что даже
синтетические голоса, прогнанные через тракт устройства, **поднимают** recall. fish.audio даёт
**100k+ разных русских голосов** за копейки («захааар» ≈ 14 байт, $15/1M байт), включая
**детские** (известное слабое место) — это самый дешёвый способ добрать разнообразие голосов.

- **drawn «захааар»** → позитивы (целевое протяжное слово);
- **short «захар»** → негативы (короткое имя, hard-негатив для различения длины).

> ⚠️ ВАЖНО: на выходе — чистые студийные wav. Сырая синтетика обучению **вредит** (v11b);
> помогает только запись через **тракт устройства**. Поэтому весь вывод ОБЯЗАТЕЛЬНО прогнать
> playback-recapture через девайс ПЕРЕД обучением. drawn → позитивы, short → негативы.

## Пайплайн (воронка)

```
fish_catalog.py   ──>  fish_catalog_ru.jsonl      весь ru-каталог (115 861 голос) + статистика
       │
fish_dedup.py     ──>  fish_voices_clean.tsv      минус клоны-сущности = 107 696 чистых голосов
  (--exclude fish_entities_confirmed.txt)
       │
gen_fish_positives.py ──> samples/positive_fish_src/     drawn «захааар» (позитивы)
  (--voices-file ...)      samples/negative_short_fish_src/  short «захар» (негативы)
       │
stt_sort.py       ──>  recognized / not_recognized   контроль качества генерации (STT)
       │
[ playback-recapture через устройство ]  ──>  обучение microWakeWord
```

## Как отбирали голоса

1. **Выкачали весь ru-каталог** — 115 861 голос (`fish_catalog.py`).
2. **Энтити-дедуп** (`fish_dedup.py`): в каталоге верхушку по популярности занимают
   **клоны селеб/персонажей** под разными названиями («путин» / «голос путина» / «В.В. Путин» /
   «Vladimir Putin» — одна сущность, четыре строки). Это раздувает выборку и режет разнообразие.
   - **Генерик-автоимена** («Молодой Женский Голос» ×2336) — НЕ клоны: это разные авторы,
     каждый загрузил свой голос = реальное разнообразие → **оставляем**.
   - **Сущности** (Кейн/Путин/Зеленский/Хоумлендер/персонажи Цифрового цирка и т.д.) →
     оператор отсмотрел рейтинг глазами и собрал **251 маску** в `fish_entities_confirmed.txt`.
   - Решение: **выкинуть всех под масками целиком** (8 165 голосов) — для генерации чище брать
     из 107 696 разных голосов, чем из 148 клонов Путина.
3. **Чистый пул** → `fish_voices_clean.tsv` (107 696: м 57k / ж 40k; **детских 3 813**).

## Рецепт генерации (выводы из прослушиваний)

- **Модель синтеза: s2-pro** (v2). s1 (v1) коверкает «захар» — не используется (дефолт исправлен).
- **Протяжность: a2…a9** (число «а» в тексте «зах»+«а»*N+«р»). На слух уже a3 — явный распев;
  a2 по длине ≈ короткий «захар», но оставлен для разнообразия. Длина растёт с числом «а»
  НЕлинейно (fish не тянет гласную пропорционально буквам) — но на слух все a2–a9 протяжные.
- **Интонации: каждый голос × все 5 тегов** (нейтрально / emphasis / angry / laughing / sobbing)
  для drawn-позитивов — разнообразие просодии.
- **Короткие негативы — ТОЛЬКО нейтрально**: sobbing/laughing растягивают «захар», и он перестаёт
  быть коротким (видели: ~15% коротких с эмоцией выходили длинными).
- **Вычитка голосов**: каждый кандидат получает короткую пробу «захар», прогоняется через
  large-v3, и голоса с кривым произношением отсеиваются ДО генерации.
- **STT-постфильтру можно доверять** (large-v3): ошибок единицы на сотню; брак в основном из-за
  кривых ГОЛОСОВ, а не интонаций.

## Скрипты

### `fish_catalog.py` — выкачать и проанализировать каталог
```bash
python fish_catalog.py --language ru                 # пулл + отчёт (jsonl рядом)
python fish_catalog.py --analyze-only fish_catalog_ru.jsonl   # только переотчёт
```
Тянет постранично (page_size 1000, ретраи на таймаут), пишет JSONL (по голосу на строку),
печатает: всего, train_mode, пол, возраст, **детские**, дедуп имён.

### `fish_dedup.py` — энтити-дедуп голосов
```bash
# 1) рейтинг кандидатов-сущностей (смотреть глазами):
python fish_dedup.py fish_catalog_ru.jsonl --min-voices 12 --top 60
# 2) полный markdown-отчёт со всеми кластерами:
python fish_dedup.py fish_catalog_ru.jsonl --report review.md
# 3) дрилл по одной сущности (все алиасы по task_count):
python fish_dedup.py fish_catalog_ru.jsonl --search путин
# 4) ИТОГ: выкинуть всех под масками -> чистый пул:
python fish_dedup.py fish_catalog_ru.jsonl \
    --exclude fish_entities_confirmed.txt --export-clean fish_voices_clean.tsv
```
Дескрипторы (молодой/женский/драматический…) гасятся стоп-листом; имена-сущности кластеризуются
по префиксу слова (ловит склонения путин→путина); короткие маски (≤3, напр. «жд») матчатся как
целое слово, длинные — подстрокой. `--exclude` печатает, какие маски выкинули больше всего
голосов (ловить слишком широкие).

### `fish_entities_confirmed.txt` — маски сущностей
Список подтверждённых оператором сущностей-клонов (по строке на маску, `#` — комментарий).
Матч: регистронезависимо, ё==е, `*` в конце игнорируется. Чтобы добавить/убрать — правишь файл
и перегоняешь `fish_dedup.py --exclude ... --export-clean ...`.

### `gen_fish_positives.py` — генерация
```bash
# полный прогон из чистого пула (на Linux-ноде: large-v3 вычитка без перегрева CPU):
python gen_fish_positives.py \
    --voices-file fish_voices_clean.tsv --voices 400 --mode both \
    --out ../samples/positive_fish_src --out-short ../samples/negative_short_fish_src

python gen_fish_positives.py --dry-run --voices 5 --voices-file fish_voices_clean.tsv  # ничего не тратит
```
Ключевое: `--voices-file` (брать из пула, а не из живого каталога), `--mode both|drawn|short`,
`--voices N` (сколько ХОРОШИХ голосов после вычитки), `--a-min/--a-max` (протяжность, дефолт 2..9),
`--no-verify` (отключить вычитку голосов), `--model` (дефолт s2-pro). Ключ API — из
`data/config.json` (`tts.instances.fishaudio.api_key`) или `$FISH_API_KEY`.

### `stt_sort.py` — контроль качества генерации
```bash
python stt_sort.py <dir-or-wavs...> --out sorted/                 # mlx GPU (Mac), быстро
python stt_sort.py <dirs...> --out sorted/ --backend faster       # large-v3 на Linux-ноде
```
Раскладывает клипы в `recognized/` и `not_recognized/` по тому, услышал ли STT «захар»-семейство;
в имя файла кладёт сырую расшифровку + пишет `manifest.tsv`.

### `mlx_stt.py` — быстрый локальный STT (Apple Silicon)
```bash
python mlx_stt.py somedir/ --model turbo   # GPU/ANE, ~1с/клип, не грузит CPU
```
Для быстрых проверок на Mac. Тяжёлые прогоны large-v3 — на Linux-нодах (там mlx нет, faster-whisper).

# ТЗ: ночная программа тренировок «захааар» — раунд РЕАЛЬНЫХ device-позитивов

## Контекст (коротко)
Wake word — протяжное русское **«захааар»** (короткий «захар» НЕ цель) на microWakeWord
(INT8 streaming TFLite, ESP32-S3 / HA Voice PE, ESPHome `micro_wake_word`,
micro_speech 40-band, feature_step_size 10).

Прошлый итог: **v11** (v8-рецепт + РЕАЛЬНЫЕ device-tract негативы тишины/музыки) убрал
ложные в тишине (silence-FAPH 12.5→0.8/ч, music 9.3→0) и улучшил FRR на живых людях
(4.7→0.9%), НО его **recall просел в поле** (dev_heldout FRR 8.5→12.4%, недодетект живого
«захааар») — поэтому в проде временно откатан на v8.

## ГЛАВНОЕ НОВОЕ В ЭТОМ РАУНДЕ
Впервые есть **РЕАЛЬНЫЕ device-tract ПОЗИТИВЫ** — записаны через сам тракт устройства
(playback re-recording + живые), на ПРАВИЛЬНОМ канале (ch0, который слушает mWW). Это
ровно тот актив, что стоял в реестре как «⏳ ЖДЁТ DEVICE-СЕМПЛОВ» и который должен
**починить просадку recall v11**. Плюс пользователь добавил реальные музыкальные негативы.

**Датасет этого раунда — ТОЛЬКО записи через устройство.** Без синтетики (yandex) и без
старых не-device позитивов. Архив `~/zakhar_samples.tgz` → распаковать → `samples/`:
- **positive_mix_recorded** (1285) — РЕАЛЬНЫЕ device-tract «захааар», 3 громкости
  (v0.3≈397 / v0.6≈445 / v1.0≈443). v0.3 прошли ASR-фильтр large-v3 + ручную вычитку
  (чистые). v0.6/v1.0 — как есть (их ASR-вычитка идёт на другой ноде, не блокирует).
- **negative_silence_recorded** (382, ~4ч) / **negative_music_recorded** (145, новая) /
  **negative_speech_recorded** (477, реальная ТВ-речь) / **negative_noise_vacuum_recorded**
  (117) — все РЕАЛЬНЫЕ через устройство. Плюс твои большие негатив-банки
  (HF/MUSAN/ESC-50/FMA), как в v8/v11.

НЕ использовать в обучении: yandex-синтетику и старый positive_samples_real_people
(не device-tract). real_people(107) держи только как ЭВАЛ-референс для real-FRR (как раньше).
v11b уже показал: добавление синтетики УХУДШАЛО real-FRR (6.5% vs 0.9%) — поэтому строим
датасет исключительно из device-записей.

## КРИТИЧНО: leakage-safe split (иначе метрики врут)
В `positive_mix_recorded` ОДИН исходный клип проигрывался на 3 громкостях → 3 записи с
ОДНИМ источником. Имя файла: `section_1_ru.<uuid>__v0.6__dev.wav` или
`yx_<voice>_<style>_NN_vX.X__v0.6__dev.wav`. **Ключ источника = всё до `__vX.X__`.**
- Делить train/held-out **ПО КЛЮЧУ ИСТОЧНИКА**, держа все 3 громкости одного источника в
  ОДНОМ фолде. Случайный сплит по файлам = утечка (v0.3 клипа X в train, v0.6 того же X в
  test). На этом уже горели (uuid-leakage в v5).
- Held-out ~25–30% по источнику. Те же live-голоса/сессии тоже не разносить по фолдам.

## Честный eval (как в v11)
Для КАЖДОЙ версии — таблица на реальном device-eval, **БЕЗ VAD** (робастность модели) и
с VAD, @ probability_cutoff 0.9 / sliding_window 5:
`silence-FAPH/ч | music-FAPH/ч | FRR real_people | FRR dev_heldout`.
Лучшую модель отгрузить как drop-in (json+tflite, INT8 streaming, влезает в ~32k пар./45КБ).

## ПРОГРАММА (каждая идея = ОТДЕЛЬНАЯ ВЕРСИЯ output_vN, по приоритету)

### 1. PRIMARY — v11-рецепт + новые реальные позитивы
v11 (реальные silence/music негативы, sampling 10/8, penalty 8, drawn-out центрирование,
confusables с/з) **+ добавить positive_mix_recorded в позитивы**. Цель: вернуть recall
(починить просадку v11), сохранив silence/music-FAPH низкими. **Это главный кандидат на
прод.** Сравнить с v8 и v11 на одном eval.

### 2. Абляция состава реальных негативов
По очереди убирать silence / music / speech / vacuum и смотреть, как двигается
соответствующий FAPH и не роняется ли recall (особенно speech-негативы). Цель — понять
минимально достаточный набор реальных негативов. Позитивы всегда только device-tract.

### 3. Абляция громкости v0.3
С/без тихих v0.3-позитивов. Тихие device-позитивы помогают обобщению (far/тихий голос)
или добавляют путаницы в тишине? Смотреть и recall, и silence-FAPH.

### 4. RE-OPEN: настоящий angular-margin head + РЕАЛЬНЫЕ музыкальные негативы
В v10 angular был САМЫМ сильным рычагом recall (worst-FRR −13…−16пп, INT8 без потерь), но
рос music-FAPH (≥9/ч) → гейт «+recall без +FAPH» не прошёл, и его ОТЛОЖИЛИ ровно «до
реальных музыкальных негативов (s≈10, m=0.2)». **Теперь они есть.** Обучить angular-голову
С реальными music/silence-негативами. Если music-FAPH удержится ≤ v11 — это, возможно,
лучшая модель по recall. Тулчейн: `research/am_train.py`.

### 5. angular + 2-ступенчатый verifier (линия v13) на новых данных
v13 офлайн выглядел лучшим, но запечённый деплой не бил v11 (silence 6.7/ч). Перепроверить
с новыми реальными позитивами/негативами — сдвинулось ли.

### 6. NEW — целевые confusables-негативы из «соседей» захар (на основе ASR-находки)
Прогон large-v3 по нашим записям показал, на что фонетически путается тихий «захар»:
**«закон», «закат», «закрой», «зачем», «заходи», «сахар», «саша», «сатана»**. Намайнить
такие слова из СВОИХ реальных речевых негатив-банков и поднять им вес как hard-confusables —
заточить границу с/з и «протяжный захар vs короткие соседи». (Запись таких слов через само
устройство сейчас недоступна на ноде → пометить pending на следующий раунд; брать из
реальных речевых корпусов, НЕ TTS-синтетику.)

### 7. Тюнинг рабочей точки лучшей модели
Sweep probability_cutoff × sliding_window × длительность-центрирование на реальном eval —
выбрать операционную точку (recall vs FAPH).

### 8. Если останется ночь
Curriculum multi-condition (clean→reverb→media по нарастанию, БЕЗ полногромкого
music-mix); стратифицированный eval по 9 акуст. режимам.

## НЕ ТРАТИТЬ НОЧЬ (уже отброшено — см. HYPOTHESIS_REGISTRY)
energy/RMS-гейт и SNR-адаптивный порог (ВРЕДЯТ: ложные в тишине AGC-громче слов);
feature-KD/logit-KD (хуже; нужно >400k поз.); focal loss; margin-BCE; residual connections;
negative_class_weight 30; adversarial mining раунды 2–3 (плато); тяжёлые SOTA (Mamba/KWT —
не влезают); **синтетические** silence/music/music-mix негативы (бесполезны/вредны — корень
в РЕАЛЬНОЙ текстуре комнаты и в AEC-loopback residual). QAT не нужен — PTQ без потерь.

## Деплой-инварианты
INT8 streaming, micro_speech 40-band, feature_step_size 10, ~32k пар./≤45КБ, VAD on в
проде. Confusables с/з (sampling 8 / penalty 5) как раньше.

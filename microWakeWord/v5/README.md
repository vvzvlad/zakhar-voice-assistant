# v5 — НОВАЯ РЕКОМЕНДОВАННАЯ модель «захааар» (drawn-out-центрированная)

Цель пересмотрена оператором: активирующая фраза — **протяжный «захааар»** (долгая «а»);
короткий «захар» НЕ цель. Основная метрика — recall на протяжной форме. Полный отчёт —
[model/TRAINING_REPORT.md](model/TRAINING_REPORT.md).

## Вердикт: v5 — основная модель

v5 — лучший по основной метрике (drawn-out recall) и **строго бьёт v4 в деплое (с VAD)**.

| метрика | v4 | **v5** |
|---------|----|--------|
| Drawn-out recall (per-clip, %≥0.9) | 99% (0.996) | **100% (0.999)** |
| «Сахар» near-miss (срабатываний из 90) | 3/90 | **1/90** |
| Музыка-FAPH **с VAD** (все cutoff) | 0.00 | **0.00** |
| Recall @ cutoff 0.9/win3 (чистая музыка) | 0.882 | **0.946** |

- **С VAD (рекомендованный деплой):** music-FAPH = 0 у обоих → решает recall, а он у v5 выше
  на каждом пороге → строгий выигрыш.
- **Без VAD:** v5 лучше в реалистичной зоне (music-FAPH 2.0–2.67 → recall 0.91–0.95 vs 0.88);
  v4 достаёт лишь крайний низко-FAPH угол (1.33 при recall ~0.80), который под VAD неактуален.
  Т.е. без VAD — выгодный размен, но не абсолютное доминирование во всём диапазоне.

## Что сдвинуло фронтир

1. **Adversarial false-trigger mining** (Amazon, arXiv 2010.06659): прогон v4 по аудио без
   «Захар» → 1313 hard-negs из **вокальной музыки** (остаток, что VAD пропускает) + 940 из
   с/з-богатой речи (семья «Сахар»); влиты с penalty 8.
2. **Drawn-out центрирование позитивов**: 528 реальных + 1056 тайм-стретч (×1.4–1.8, питч
   сохранён) + 7247 удлинённой синтетики (Piper+Silero, «захааар/захаааар/захаааааар»).
   Короткий «захар» убран. Это подняло drawn-out recall до 100% и срезало «Сахар».
3. **Длительность/окно как дискриминатор**: длинная форма держит устойчивую вероятность,
   короткие ложные (Сахар, музыкальные всплески) — нет. clip_duration 1800 мс.
4. **VAD-прегейт** (из v4): инструментальная музыка → 0 ложных.

## Как ставить

`probability_cutoff 0.95` / `sliding_window_size 3`, `tensor_arena 45000` (сверь с живым
JSON). **Обязательно `vad:`** в ESPHome (см. [model/esphome_micro_wake_word.yaml](model/esphome_micro_wake_word.yaml)).

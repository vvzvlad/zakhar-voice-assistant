## Цель и жёсткие ограничения (факты)
On-device wake word — протяжный «захааар» (рус., долгая «а»; короткий «захар» НЕ цель) на
microWakeWord (OHF-Voice/micro-wake-word): TensorFlow → INT8 streaming TFLite для TFLM на
ESP32-S3 (Home Assistant Voice PE), ESPHome `micro_wake_word`. Модель ~32k параметров, ~78 КБ,
вход 16 кГц моно, фронтенд micro_speech 40-band (feature_step_size 10) — ФИКСИРОВАН. tensor_arena
~45 КБ. Обучение CPU-only. Фраза фиксирована. Перед моделью — XMOS XU316 (AEC/NS/AGC) + TF4micro;
есть встроенный VAD-прегейт.

## Текущее состояние (v8, продакшен) — факты
Обучена на чистых синтетических позитивах + реальных записях, прошедших аудиотракт устройства.
Оценка на leakage-safe device-eval (реальный тракт): on-device FRR **~5%** (recall 0.95),
device-FAPH **0/час с VAD**. На 40-мин музыкальном стриме при cutoff 0.95/окно5: 0 ложных/час.
Деплой: cutoff 0.95, sliding_window_size 5, VAD on.

## Что уже пробовали — и результаты (НЕ предлагать заново)

### Версии
| версия | рычаг | результат |
|--------|-------|-----------|
| v1 | baseline drawn-out синтетик + 621 real | рабочая, music-FAPH ~8/ч noVAD / 0 VAD |
| v2 | negative_class_weight 30 | регресс (recall упал) |
| v3a/v3b | выделенные муз-негативы; крупнее модель (5-блок 32k) | не побили v1 |
| v4 | масштаб негативов (HF+MUSAN+ESC-50+FMA) + confusables + Silero | recall 99%, music-FAPH 2.67 noVAD |
| v5 | drawn-out центрирование + adversarial mining + рычаг окна/длительности | лучшая до v8 (eval тёк по uuid) |
| v6/v7 | adversarial mining раунды 2–3 | плато |
| **v8** | чистый синтетик + реальные device-капчи + far-field RIR + uuid-safe split | **продакшен, device-FRR 5%** |

### Отдельные техники (программа v9; всё гейтилось на device-eval)
| техника | результат |
|---------|-----------|
| knowledge distillation (teacher→student) | без выигрыша на нашем масштабе данных |
| 2nd-stage verifier (логрег на bottleneck) | РАБОТАЕТ (режет no-VAD муз-ложные), готов к деплою |
| QAT | не нужен — INT8 PTQ уже без потерь |
| focal loss (γ=2) | регресс |
| margin-BCE (logit-shift аппрокс AM-softmax) | регресс |
| residual connections | вредят |
| RIR-банк OpenSLR SLR28 | вредит (слишком гулкий vs mit_rirs) |
| агрессивная multi-condition аугментация | пере-портит (плато ~20×) |
| арх-свип BC-ResNet / TC-ResNet | не побили mixednet |
| софт-микс музыки в обучение | вредит (см. находку 1 ниже) |

### Что СРАБОТАЛО (данные/методы)
- VAD-прегейт → инструментальная музыка 0 ложных.
- **Playback re-recording** (запись синтетики/реальных через динамик→микрофон устройства) →
  закрыл train/serve gap, FRR 18%→5%. Главный рычаг.
- STT-фильтр синтетики (нашли 62% мусора от сломанного TTS, починили).
- uuid-disjoint сплит + device-eval (устранили утечку, завышавшую recall).
- Длительность как дискриминатор (протяжная фраза + большое окно).

## Открытые проблемы (факты, без наших решений)
1. **Ложные срабатывания в реальной ТИШИНЕ/idle-комнате.** При этом синтетика баг НЕ
   воспроизводит: на смоделированном шуме (silence/white/pink/AGC-boosted/syllable-modulated)
   обе модели дают 0.0 ложных при всех порогах. Реальные ложные приходят с живой текстуры комнаты.
2. **Худшие режимы: одновременная речь+медиа, far-field, приглушённость.** Попытка
   смоделировать канал в обучении дала РЕГРЕСС: устройство гонит AEC с loopback-референсом
   собственного медиа → модель видит остаток эха, а не полный сигнал; синтетический микс — не то
   распределение.
3. **Модель-сайд (по нашим прогонам) исчерпан:** distillation/QAT/loss/архитектура — регресс или
   0; реальный прирост (v5→v8) дали только реальные данные через тракт.
4. Средние метрики ~на уровне индустрии; неясно, где реальный потолок 2-сложной фразы.

# v9 — model/engine breadth-программа (НИЧЕГО НЕ ОТГРУЖЕНО; v8 остаётся продакшеном)

~10-часовая автономная программа модельных/движковых экспериментов, пока ждём реальные
device-записи long-tail. Полный отчёт — [TRAINING_REPORT_v9.md](TRAINING_REPORT_v9.md).

## Вердикт: v8 остаётся, регресс не отгружён

**Ни один модельный кандидат не побил v8** на leakage-safe device-eval (стратифицированном по
режимам). Это контролируемый отрицательный результат: v8 — сильный локальный оптимум под
текущие данные, и главный рычаг теперь — **реальные полевые данные**, а не правки
архитектуры/loss/аугментации.

| кандидат | рычаг | dev-FRR@0.9 | вердикт |
|----------|-------|------------:|---------|
| **v8 (инкумбент)** | mit_rirs, без residual | **0.059** | **остаётся (не побит)** |
| v9-multicond | SLR28 far-field + музыка, агрессивно | 0.098 | регресс (пере-аугментация) |
| v9-margin | margin-BCE (AM-softmax approx) | 0.111 | регресс |
| v9-focal | focal γ=2 | 0.144 | регресс |
| v9-res | residuals + multicond | 0.085 | регресс |
| v9-distill | teacher→student KD | 0.170 | регресс |

## Чему научились (чистые атрибуции)
1. **Residuals вредят** на этих данных (0.065→0.137); прежний «residuals помогают» был шумом.
2. **SLR28 RIR-банк вредит** — его комнаты слишком гулкие для ближнеполевого устройства; mit_rirs лучше.
3. **Агрессивная аугментация пере-портит** позитивы (подтверждает плато ~20×).
4. **KD без выигрыша** на нашем масштабе (~10k синт + ~900 real; нужно >400k позитивов).
5. **focal/margin регрессируют** — штатный weighted-BCE лучший; настоящий angular-margin head — правильный порт, если возвращаться к loss на реальных данных.
6. **QAT не нужен** — INT8 PTQ уже без потерь vs float (≤0.7 pp).
7. v8 лучший в КАЖДОМ режиме; худшие у всех: `music_snr5`, `reverb+music10`, `muffled` — цели для реальных данных.

## Единственное полезное к отгрузке: 2-я стадия-verifier
[research/G2_VERIFIER.md](research/G2_VERIFIER.md), [verifier/](verifier/): логрег 65 параметров
на 64-мерном bottleneck KWS, запуск только после триггера. На held-out: 100% истинных wake-ов,
**отвергает 100% намайненных no-VAD музыкальных ложных** (~260 байт, ~0 латентности). Комплемент
к v8 без регресса — ценен для no-VAD / low-latency сценариев.

## Ключевой forward-инсайт (важно для завтрашней записи)
Доп. ресёрч (Amazon 1808.00563, Opatka IS-2021): **софт-микс музыки — НЕВЕРНОЕ распределение для
barge-in**. HA Voice PE гоняет AEC с loopback-референсом проигрываемого ИМ медиа → KWS видит лишь
**ОСТАТОК эха**, а не полногромкую музыку. Поэтому синтетический `music_snr5` пере-портил обучение,
а eval ПЕРЕоценивает проблему. **Снимать завтра нужно реальный остаток:** «Захар» произнесённый,
ПОКА устройство играет СВОЙ TTS/медиа (через настоящий AEC) + far-field/muffled в реальной комнате.
Синтетикой это не воспроизвести. Runbook — [REALDATA_RUNBOOK.md](REALDATA_RUNBOOK.md).
